UI Reactivity Test

Instructions

  1. Open the application in your browser (http://localhost:5173)
  2. Open the Products page in one tab
  3. Open the Admin panel in another tab (or same tab)
  4. In the Admin panel, create, edit, or delete a product
  5. Switch back to the Products page tab
  6. Verify that the Products page has updated automatically without refreshing

Manual Test in Console

You can also test the event system directly in the browser console:

// Simulate a product update
const testProducts = [
  {
    id: "test-1",
    name: "Test Product",
    description: "This is a test product",
    imageUrl: "",
    variants: []
  }
];

// Save products (this will trigger the event)
localStorage.setItem('gelkrupa_products', JSON.stringify({ products: testProducts }));
window.dispatchEvent(new CustomEvent('productsUpdated', { 
  detail: { products: testProducts } 
}));

Expected Behavior

  • ✓ Products page loads initial data on mount
  • ✓ Admin panel can create, edit, and delete products
  • ✓ When Admin panel saves changes, a custom event is dispatched
  • ✓ Products page listens for the custom event
  • ✓ Products page updates its state when the event is received
  • ✓ Changes are reflected immediately without page refresh

Implementation Details

Custom Event System:

  • saveProducts() in storage.js dispatches a 'productsUpdated' event
  • Products page listens for 'productsUpdated' events in useEffect
  • Event handler updates the products state with new data
  • Event listener is cleaned up on component unmount

Cross-Tab Communication

Note: The current implementation works within the same browser window/tab. For cross-tab communication (when Products and Admin are in different tabs), you would need to use the 'storage' event:

// In Products.jsx, add this to useEffect:
window.addEventListener('storage', (e) => {
  if (e.key === 'gelkrupa_products') {
    const products = getProducts();
    setProducts(products);
  }
});

However, the 'storage' event only fires in OTHER tabs, not the one that made the change. The custom event handles same-tab updates.
